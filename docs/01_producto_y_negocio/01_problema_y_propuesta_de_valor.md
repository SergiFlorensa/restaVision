# Problema y propuesta de valor

## Dolor operativo principal
En hostelería, el responsable de sala debe decidir en segundos con información imperfecta:
- aceptar o no a un grupo sin reserva,
- prometer un tiempo de espera razonable,
- anticipar liberación de mesas,
- detectar fricciones y pérdidas.

## Qué aporta el sistema
### Para el director / responsable de sala
- menor incertidumbre,
- decisiones más rápidas,
- menos saturación mental,
- mejor visión del estado real del local.

### Para el restaurante
- mejor rotación de mesas,
- menor tiempo de espera mal prometido,
- menor abandono de cola,
- potencial reducción de pérdidas invisibles,
- base de datos operativa para mejora continua.

### Para una cadena
- estandarización de criterios,
- comparación entre sedes,
- análisis por franja y local,
- escalabilidad técnica y de negocio.

## Propuesta de valor resumida
“Copiloto visual y predictivo para la gestión de sala en tiempo real, construido con tecnología open source y pensado para operar localmente.”

## Diferenciación
La mayoría de soluciones de reservas o waitlist trabajan con estados manuales o semimanuales. Este proyecto busca reducir la fricción captando señales directamente de la sala.

## Modelo comercial potencial
- licencia por local,
- instalación y personalización,
- mantenimiento,
- dashboards multi-sede,
- módulo premium de alertas / predicción.

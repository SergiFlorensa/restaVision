# ADRs iniciales (Architecture Decision Records) resumidos

## ADR-001 — Procesamiento local por defecto
Decisión:
- realizar inferencia local/edge para el MVP.

Motivo:
- latencia,
- privacidad,
- independencia de nube,
- menor coste recurrente.

## ADR-002 — Open source primero
Decisión:
- elegir herramientas abiertas siempre que no bloqueen el objetivo.

Motivo:
- TFG,
- control de costes,
- libertad de iteración.

## ADR-003 — MVP doméstico antes que piloto real
Decisión:
- validar el pipeline en una mesa en casa antes del restaurante.

Motivo:
- coste bajo,
- iteración rápida,
- menor complejidad.

## ADR-004 — Baselines simples antes que modelos complejos
Decisión:
- construir primero reglas y modelos clásicos.

Motivo:
- explicabilidad,
- rapidez,
- menor necesidad de datos.

## ADR-005 — Alertas discretas antes que audio conversacional
Decisión:
- empezar con dashboard + sonido suave, no con agente hablador continuo.

Motivo:
- reducir saturación,
- simplificar MVP,
- mejorar adopción.

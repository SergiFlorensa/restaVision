# Criterios de Done

## Un módulo está “done” cuando:
- tiene objetivo claro,
- tiene entradas y salidas definidas,
- se puede probar,
- tiene logging mínimo,
- tiene documentación básica,
- tiene métrica asociada,
- no rompe la latencia objetivo del sistema.

## Ejemplo
### Detección de ocupación
Done cuando:
- detecta mesa libre/ocupada en pruebas definidas,
- registra eventos,
- permite depuración visual,
- documenta errores típicos.
